# DyLoRA: Parameter-Efficient Tuning using Dynamic Search-Free Low-Rank Adaptation

> **论文信息**：Valipour et al., EACL 2023  
> **一句话概括**：标准 LoRA 需要预先固定秩 $r$，如果选错则需要重新训练。DyLoRA 在训练过程中**同时训练所有 $r' \leq r$ 的 LoRA**——训练一次，推理时可以用任意 $r' \leq r$ 的秩，无需重新训练。

## 贯穿全文的例子

> 不确定目标任务需要多大的 $r$。传统做法：分别训练 $r=4, 8, 16, 32, 64$ 五个模型 → 评估 → 选最好的。成本是 5 倍。
>
> **DyLoRA**：训练一次（$r_{\max}=64$），得到一个模型。推理时可以截取 $r=4, 8, 16, 32, 64$ 中任意一个，每个都有竞争力。

---

## 一、核心思想

### 1.1 嵌套结构

DyLoRA 利用了一个观察：如果用 $r=64$ 训练的 LoRA，其前 $r'=8$ 个秩分量是否就等于用 $r=8$ 训练的 LoRA？

**答案通常是"不"**——标准 LoRA 训练时所有 64 个分量同时优化，前 8 个分量不会特别好用。

**DyLoRA 的解决方案**：训练时随机采样不同的 $r'$，强制每个 $r' \leq r_{\max}$ 都能工作：

$$
\text{每步训练}：r' \sim \text{Uniform}(1, r_{\max})
$$

只使用 $A_{1:r', :}$ 和 $B_{:, 1:r'}$ 进行前向和反向传播。

### 1.2 训练算法

```
for each training step:
    1. 采样秩: r' ~ Uniform(1, r_max)
    2. 截取参数: A' = A[0:r', :], B' = B[:, 0:r']
    3. 前向传播: h = W₀x + B'A'x
    4. 计算 loss 并反向传播
    5. 只更新 A[0:r', :] 和 B[:, 0:r'] 的梯度
```

### 1.3 为什么有效？

通过在不同 $r'$ 间随机切换训练：
- 每个秩分量都会被单独训练到
- 前 $r'$ 个分量需要在没有后续分量帮助时独立完成任务
- 这强制了一种"重要性排序"：前面的分量比后面的更重要

---

## 二、实验结果

在 RoBERTa-Base 上：

| 推理时的 $r$ | 单独训练 LoRA | DyLoRA (一次训练) | 差距 |
|-------------|--------------|------------------|------|
| 4 | 86.2 | 85.8 | -0.4 |
| 8 | 87.1 | 86.9 | -0.2 |
| 16 | 87.5 | 87.3 | -0.2 |
| 32 | 87.8 | 87.6 | -0.2 |
| 64 | 87.9 | 87.7 | -0.2 |

DyLoRA 在所有秩上都接近单独训练的效果（差距 < 0.5%），**但只需训练一次**。

---

## 三、适用场景

- **不确定最优 $r$ 时**：训练一次即可探索
- **不同设备需要不同计算预算**：大 GPU 用 $r=64$，小 GPU 用 $r=8$
- **渐进式部署**：先用小 $r$ 快速响应，后台切换到大 $r$

---

## 四、总结

DyLoRA 是一种**训练一次、多秩使用**的方案，代价是每个固定秩的效果略低于单独训练。适合需要灵活部署的场景。
