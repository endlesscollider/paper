## 配置一：ManiSkill PPO + OpenVLA（入门）

### 场景特点
- 单机 2 卡快速验证
- 用 LoRA 微调 OpenVLA-7B
- GPU 显存紧张 → 开启 offload

### 关键配置解读

```yaml
cluster:
  component_placement:
    actor,env,rollout: 0-1    # 三组共享 2 张 GPU

actor:
  enable_offload: True        # 必须开！2 张 GPU 装不下两个 7B 模型
  model:
    is_lora: True             # LoRA 微调，减少参数量
    add_value_head: True      # PPO 需要 value head

rollout:
  enable_offload: True        # 推理完就 offload
  pipeline_stage_num: 2       # 用 pipeline 藏延迟

env:
  train:
    total_num_envs: 32        # 32 个并行环境（ManiSkill GPU 渲染快）
    max_episode_steps: 80     # 短 episode
```

### 为什么选这些参数？

- `is_lora: True`：7B 模型全量微调需要 ~80GB 优化器状态，LoRA 只需 ~1GB
- `pipeline_stage_num: 2`：环境和推理交替用 GPU，pipeline 让它们并行
- `max_episode_steps: 80`：ManiSkill 的抓取任务 80 步够了

## 配置二：LIBERO GRPO + Pi0（主流方案）

### 场景特点
- 8 卡训练
- 稀疏奖励（只有成功/失败）→ GRPO
- Pi0 模型不支持 gradient_checkpointing

### 关键配置解读

```yaml
algorithm:
  group_size: 8               # 每个任务跑 8 次
  rollout_epoch: 8            # 8 轮采集
  filter_rewards: True        # 过滤全成功/全失败
  logprob_type: chunk_level   # Pi0 的 logprob 是 chunk 级别

actor:
  micro_batch_size: 128       # 无 value head，显存充裕
  global_batch_size: 2048     # 大 batch 稳定训练
  model:
    add_value_head: False     # GRPO 不需要
  fsdp_config:
    sharding_strategy: "no_shard"  # Pi0 3B，单卡放得下
    gradient_checkpointing: False  # Pi0 不支持

rollout:
  enable_offload: False       # 8 卡充裕，不需要 offload
  pipeline_stage_num: 1       # 不共享 GPU，不需要 pipeline

env:
  train:
    total_num_envs: 64        # 64 / group_size 8 = 8 个不同任务
    group_size: ${algorithm.group_size}
    max_episode_steps: 240    # LIBERO 长 episode
```

### 为什么选这些参数？

- `group_size: 8`：LIBERO 奖励只有 0/1，需要组内对比才能有 gradient signal
- `sharding_strategy: no_shard`：Pi0 约 3B，一张 80GB A800 足够
- `filter_rewards: True`：全成功组的 advantage 全 0（无信息），跳过

## 配置三：IsaacLab PPO + GR00T（大规模）

### 场景特点
- 4096 个并行环境（IsaacLab GPU 加速）
- 专用 GPU 分配：Actor 和 Env 分开
- GR00T N1.5 模型

### 关键配置解读

```yaml
cluster:
  component_placement:
    actor: 0-3               # Actor 独占 GPU 0-3
    env,rollout: 4-7         # Env 和 Rollout 共享 GPU 4-7

env:
  train:
    total_num_envs: 4096     # IsaacLab 支持大规模并行
    max_episode_steps: 200

actor:
  micro_batch_size: 64
  global_batch_size: 4096    # 和环境数匹配
  model:
    model_type: gr00t
    num_action_chunks: 16    # GR00T 一次出 16 步
  enable_offload: False      # 独占 GPU，不需要 offload
```

### 为什么选这些参数？

- `actor: 0-3`：Actor 独占 4 张 GPU，可以做 FSDP full_shard
- `total_num_envs: 4096`：IsaacLab 用 GPU 做物理仿真，可以跑上千个并行
- `num_action_chunks: 16`：GR00T 的预训练设计就是 16 步 chunk

## 配置四：Async PPO + Pi0.5（高吞吐）

### 场景特点
- 异步模式最大化 GPU 利用率
- 需要处理 off-policy 偏差

### 关键配置解读

```yaml
# 异步模式的 Runner 选择通过 loss_type 和训练入口决定
# 使用 async_embodied_runner.py 而非 embodied_runner.py

actor:
  sync_weight_no_wait: True  # 异步权重同步（不等待完成）
  
algorithm:
  loss_type: actor_critic    # 仍然是 PPO
  # 异步模式中 decoupled PPO 使用 version 信息做修正
```

### Decoupled PPO 的配置

异步模式下自动启用 Decoupled PPO，利用 `versions` 做 importance sampling：

```yaml
algorithm:
  importance_sampling_fix: True  # 让 rollout 返回 logprobs
```

## 快速配置模板

### 最小可用配置（2 GPU 快速验证）

```yaml
cluster:
  num_nodes: 1
  component_placement:
    actor,env,rollout: 0-1

algorithm:
  adv_type: gae
  loss_type: actor_critic
  group_size: 1
  rollout_epoch: 1
  gamma: 0.99
  gae_lambda: 0.95
  clip_ratio_high: 0.2
  clip_ratio_low: 0.2

actor:
  micro_batch_size: 16
  global_batch_size: 64
  enable_offload: True
  model:
    add_value_head: True
    is_lora: True

rollout:
  enable_offload: True
  pipeline_stage_num: 2

env:
  train:
    total_num_envs: 16
    max_episode_steps: 100
```

### GRPO 标准配置（8 GPU）

```yaml
cluster:
  component_placement:
    actor,env,rollout: all

algorithm:
  adv_type: grpo
  loss_type: actor
  group_size: 8
  rollout_epoch: 8
  update_epoch: 2
  filter_rewards: True
  clip_ratio_high: 0.2
  clip_ratio_low: 0.2

actor:
  micro_batch_size: 128
  global_batch_size: 2048
  model:
    add_value_head: False

rollout:
  pipeline_stage_num: 1

env:
  train:
    total_num_envs: 64
    group_size: ${algorithm.group_size}
```

## 常见问题排查

| 症状 | 可能原因 | 解决方案 |
|------|---------|---------|
| OOM | 模型太大 / batch 太大 | 开 `enable_offload`，减 `micro_batch_size`，开 `is_lora` |
| 训练很慢 | 权重同步频繁 / pipeline_stage_num=1 | 增大 `weight_sync_interval`，增大 `pipeline_stage_num` |
| reward 全 0 | 环境配置错误 | 检查 `env_type`、`max_episode_steps` 是否合理 |
| loss 不下降 | 学习率太大/太小 | PPO 用 1e-4~5e-5，GRPO 用 5e-6~1e-5 |
| GRPO 无效 | 全成功或全失败 | 开 `filter_rewards`，调 `group_size` |
| 异步模式不稳定 | off-policy 偏差大 | 减小 `weight_sync_interval`，开 `importance_sampling_fix` |
