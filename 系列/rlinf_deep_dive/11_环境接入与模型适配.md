# 环境接入与模型适配

> 前情提要：上一章覆盖了 SAC、DAgger 等算法变体。本章讲解 RLinf 的扩展点：如何接入新环境和新模型。

## 环境注册系统

### get_env_cls() 工厂

`rlinf/envs/__init__.py` 中的 `get_env_cls()` 是环境的入口：

```python
def get_env_cls(env_type: str, env_cfg=None):
    env_type = SupportedEnvType(env_type)
    
    if env_type == SupportedEnvType.MANISKILL:
        from rlinf.envs.maniskill.maniskill_env import ManiskillEnv
        return ManiskillEnv
    elif env_type == SupportedEnvType.LIBERO:
        from rlinf.envs.libero.libero_env import LiberoEnv
        return LiberoEnv
    # ... 其他环境
```

### 环境类的接口要求

所有环境必须实现以下接口（类 Gymnasium）：

```python
class BaseEnv:
    def __init__(self, env_cfg, num_envs, ...):
        """初始化环境"""
        
    def reset(self) -> dict:
        """重置环境，返回初始观测"""
        # 返回: {"main_images": tensor, "states": tensor, "task_descriptions": list}
    
    def step(self, actions: torch.Tensor) -> tuple:
        """执行动作，返回 (obs, rewards, dones, infos)"""
        # actions: [num_envs, action_dim]
        # obs: dict（同 reset 返回格式）
        # rewards: [num_envs]
        # dones: [num_envs] bool
        # infos: dict
```

### 环境配置文件

每个环境有独立的 YAML 配置，放在 `examples/embodiment/config/env/` 下：

```yaml
# env/libero_spatial.yaml
env_type: libero
init_params:
  task_suite: spatial
  task_names: null           # null = 使用 suite 中所有任务
  image_size: [128, 128]
  use_eye_in_hand: True
  use_depth: False
  action_dim: 7             # 7-DOF: xyz + rpy + gripper
```

### 接入新环境的步骤

1. **添加枚举**：在 `SupportedEnvType` 中注册
2. **实现环境类**：继承基类，实现 `reset()` 和 `step()`
3. **写 env config**：`config/env/my_env.yaml`
4. **在 `get_env_cls()` 中注册**

### 环境 Wrapper

`rlinf/envs/wrappers/` 提供了常用的环境包装器：

| Wrapper | 功能 |
|---------|------|
| `RecordVideo` | 录制评估视频 |
| `ActionRepeat` | 动作重复（降低控制频率） |
| `FrameStack` | 堆叠历史帧 |

## 模型注册系统

### get_model() 工厂

`rlinf/models/__init__.py` 中的 `get_model()` 加载策略模型：

```python
def get_model(model_cfg):
    model_type = model_cfg.model_type
    
    if model_type == "openvla":
        from rlinf.models.embodiment.openvla import OpenVLAPolicy
        return OpenVLAPolicy(model_cfg)
    elif model_type == "openpi":
        from rlinf.models.embodiment.openpi import OpenPiPolicy
        return OpenPiPolicy(model_cfg)
    # ...
```

### BasePolicy 接口

所有具身策略模型必须继承 `BasePolicy`：

```python
class BasePolicy(nn.Module):
    def predict(self, obs: dict) -> tuple[torch.Tensor, dict]:
        """
        推理接口（Rollout Worker 调用）
        
        Args:
            obs: {"main_images": ..., "states": ..., "task_descriptions": ...}
        
        Returns:
            actions: [B, num_action_chunks, action_dim]
            result: {
                "prev_logprobs": [B, chunks],
                "prev_values": [B, 1] (可选),
                "forward_inputs": dict (用于 Actor 重算 logprobs),
            }
        """
    
    def forward(self, batch: dict) -> torch.Tensor:
        """
        训练接口（Actor Worker 调用）
        
        Args:
            batch: 包含 forward_inputs + 标签的 dict
        
        Returns:
            logprobs: [B, chunks, ...] 新策略的 log probability
        """
    
    def compute_logprobs(self, batch: dict) -> torch.Tensor:
        """计算给定动作的 logprobs"""
```

### 支持的模型一览

| model_type | 模型 | 动作类型 |
|-----------|------|---------|
| `openvla` | OpenVLA (7B) | 离散 token → 连续动作 |
| `openvla_oft` | OpenVLA-OFT | 离散 token → 连续动作 |
| `openpi` | π₀ / π₀.₅ | Flow matching → 连续动作 |
| `gr00t` | GR00T N1.5 | Diffusion → 连续动作 |
| `dexbotic_pi` | Dexbotic | Flow matching |
| `mlp_policy` | 简单 MLP | 直接输出连续动作 |
| `cnn_policy` | CNN + MLP | 直接输出连续动作 |
| `flow_policy` | Flow matching MLP | Flow → 连续动作 |
| `lingbotvla` | LingBot-VLA | Token → 连续动作 |

### 模型配置文件

`examples/embodiment/config/model/` 下每个模型一个 YAML：

```yaml
# model/pi0.yaml
model_type: openpi
model_path: null            # 运行时指定
precision: bf16
num_action_chunks: 5        # 一次预测几步动作
action_dim: 7               # 动作维度
add_value_head: False       # 是否加 value head（PPO 时设 True）
is_lora: False              # 是否使用 LoRA
```

### 添加新模型的步骤

1. **实现 Policy 类**：继承 `BasePolicy`，实现 `predict()` 和 `forward()`
2. **注册**：在 `get_model()` 和 `SupportedModel` 枚举中添加
3. **写 model config**：`config/model/my_model.yaml`
4. **适配 Rollout Worker**：确保 `predict()` 返回格式正确

### num_action_chunks 的含义

VLA 模型一次推理产出多步动作（action chunking）：

```
一次 predict() → [chunk_1, chunk_2, chunk_3, chunk_4, chunk_5]
                    ↓         ↓         ↓         ↓         ↓
                  step 1   step 2    step 3    step 4    step 5
```

`num_action_chunks=5` + `max_steps_per_rollout_epoch=240` → 一个 epoch 有 48 个 chunk steps。

## 动作处理

### action_utils.py

`rlinf/envs/action_utils.py` 中的 `prepare_actions()` 负责将模型输出转换为环境可接受的动作格式：

- **归一化/反归一化**：模型输出 [-1, 1]，环境需要真实物理量
- **旋转表示转换**：6D rotation → quaternion → euler
- **动作裁剪**：限制在合理范围内

### rotation_utils.py

处理机器人旋转的各种表示转换（rlinf/envs/rotation_utils.py）：

| 表示 | 维度 | 用途 |
|------|------|------|
| Euler (rpy) | 3 | 人类可读 |
| Quaternion | 4 | 环境接口 |
| 6D rotation | 6 | 模型输出（连续、无万向锁） |
| Axis-angle | 3 | 某些仿真器 |
