# 写作规范

## 0. 项目愿景

**本项目的终极目标**：让一个零基础的小白，完全在这一个网站上就能学会 RL、深度学习、以及机器人学习的所有相关知识。读者不需要翻论文原文、不需要找其他教程——这里就是他的唯一学习入口。

为了实现这个愿景，必须遵守以下核心原则：

### 知识自包含原则

- **遇到任何超出大学本科水平的概念，必须自动写一篇前置知识去解释它**。不能假设读者知道 KL 散度、ELBO、GAE、Actor-Critic 是什么——每一个都要有独立的前置知识文章。
- **判断标准**：如果一个概念对"有基本线代和概率知识的本科生"来说需要学习，就必须有前置知识文章。
- **执行方式**：写任何文章时，遇到需要解释的概念，如果已有前置知识就链接过去；如果没有，**立刻创建**对应的前置知识文件（哪怕先写个框架后面再填充），绝不留空白。

### 前置知识的原子性与无重复原则

- **原子化**：一篇前置知识只讲**一个**概念。"SAC"是一篇、"DDPG"是一篇、"Q 函数"是一篇、"Replay Buffer"是一篇。绝不在一篇里塞两个独立概念。
- **无重复**：同一个概念在整个项目中只有**一个**权威讲解位置。如果"Q 函数"在 SAC 文章中需要解释，不要在 SAC 文章里重新讲一遍 Q 函数——链接到 Q 函数的前置知识文章就行。
- **粒度判断**：如果一个子概念在多篇文章中被反复解释，说明它应该被提取为独立的前置知识。反之，如果一个概念只在一篇文章中出现一次且 3 句话能说清，可以内联解释不用单独成文。

### 知识图谱式的链接网络

项目的所有文章构成一个**有向无环图（DAG）**：
- 前置知识 → 前置知识（如"DDPM" → "对数似然与 ELBO"）
- 前置知识 → 论文精读（如"PPO" → "DPPO 精读"）
- 论文精读 → 综述（如"DPPO 精读" → "扩散策略 RL 综述"）

读者可以从任何一个节点进入，顺着链接探索整个知识体系，永远不会遇到"这个概念没解释"的断点。

## 1. 面向读者：小白友好 + 有深度

### 核心原则

- **假设读者是聪明的本科生**：有基本数学素养（线代、概率），但不了解具体领域。任何超出这个基线的知识（如 KL 散度、策略梯度、扩散模型、变分推断），都必须有前置知识文章兜底
- **每个公式必须"拆碎了讲"**：不能只列公式，必须逐项解释每个符号的含义、为什么是这个形式
- **先直觉再形式**：任何概念先用一句话说明"在干什么"，再给严格定义
- **举例子！举例子！举例子！**：最好有一个**贯穿全文的具体例子**（如"一个 2D 桌面上的机器人抓取"），每次引入新概念时都用这个例子来解释
- **⚠️ 必须比原论文更详细**：不论是精读还是综述，写出来的内容必须比原论文更加详细、更加易懂。原文一笔带过的地方，我们要展开讲清楚；原文没有例子的地方，我们要补充具体数值例子；原文跳步的推导，我们要补全中间步骤。**绝不允许一句话概括一个方法就跳过**——如果一个概念值得提，就值得花几段话把它讲透。这是本项目存在的意义：让读者读我们的文章就够了，不需要再回去看原文。
- **⚠️ 遇到未覆盖的概念必须主动补充**：写文章过程中，一旦发现引用了一个还没有对应前置知识的概念（如提到 DDPG 但没有 DDPG 前置文章），必须**立即创建**那篇前置知识（至少创建文件+写好骨架），不能只留 TODO 不管。项目的完整性比任何单篇文章的完成度更重要。

### 公式讲解模板

每个重要公式后必须跟随：
1. **为什么需要这个公式**：先交代动机——我们遇到了什么问题，为什么需要用数学来解决它
2. **一句话直觉**：这个公式在算什么
3. **逐项拆解**：每个符号是什么，它的物理/几何含义是什么
4. **具体数值例子**：代入实际数字算一遍（哪怕很简单的数字）
5. **为什么是这个形式**：设计动机，为什么不用其他形式

### 前置知识引用原则

- **公式中出现的通用概念**（如 $Q(s,a)$、$V(s)$、advantage、GAE、KL 散度、交叉熵等），如果不是本文主题，**必须链接到前置知识文章**
- **前置知识应该原子化**：一篇前置知识只讲一个概念（如"Q 函数与 Value 函数"是一篇、"GAE 优势估计"是一篇），但必须讲透、讲明白
- **判断标准**：如果一个符号/概念需要超过 3 句话才能解释清楚，它就应该有自己的前置知识文章
- **行文中遇到需要前置知识但还没写的文章时**：先写一个占位链接标注 `<!-- TODO: 需要前置知识文章 -->`，然后尽快补充

### ⚠️ 前置知识的独立性原则（极其重要）

**前置知识是上游，论文精读/综述是下游。** 依赖关系只能单向流动：论文可以引用前置知识，但前置知识绝不能依赖某篇论文的上下文。

具体要求：
- **前置知识必须完全独立于任何论文**：讲 SAC 就讲 SAC 本身，不要说"为了训练 Residual 网络所以用 SAC"。这种应用场景的描述属于论文精读的内容。
- **前置知识的例子必须是通用的**：用"一个机械臂学习抓取"或"一个小车学平衡"这种通用场景，不要用某篇论文特有的场景（如"VLA 的 Residual 修正网络"）
- **前置知识的动机必须是概念本身的动机**：讲 SAC 的动机是"连续动作空间需要 sample-efficient 的 off-policy 方法"，而不是"Residual RL 需要用 SAC"
- **论文精读中再做关联**：在论文精读中写"本文使用 [SAC](/前置知识/000k_前置知识_SAC_Soft_Actor_Critic) 训练 Residual 网络，具体原因是……"

**判断标准**：如果删掉项目中所有论文精读和综述文章，前置知识文章读起来仍然完整、自洽、有价值——这就对了。前置知识是**教科书式的基础概念讲解**，不是某篇论文的配套说明。

示例：

```markdown
$$
L^{\text{CLIP}}(\theta) = \mathbb{E}\left[\min\left(r_t(\theta)\hat{A}_t, \; \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon)\hat{A}_t\right)\right]
$$

**一句话**：这个损失函数让策略朝着"好动作"更新，但不允许一次更新太大。

**逐项拆解**：
- $r_t(\theta) = \frac{\pi_\theta(a_t|s_t)}{\pi_{\theta_{\text{old}}}(a_t|s_t)}$ — 新旧策略的概率比值，=1 表示没变
- $\hat{A}_t$ — 优势估计，正值说明这个动作比平均好
- $\text{clip}(r, 1-\epsilon, 1+\epsilon)$ — 把比值限制在 $[0.8, 1.2]$（$\epsilon=0.2$）
- $\min(\cdot, \cdot)$ — 取两者较小值，防止过于乐观的更新

**代入数字**：假设 $\hat{A}_t = +2$（这个动作很好），$r_t = 1.5$（新策略把这个动作的概率提高了 50%）：
- 未裁剪项：$1.5 \times 2 = 3$
- 裁剪项：$\text{clip}(1.5, 0.8, 1.2) \times 2 = 1.2 \times 2 = 2.4$
- 取 $\min(3, 2.4) = 2.4$ → 限制了过大更新

**为什么这样设计**：如果不裁剪，策略可能一步跳太远，进入不可恢复的差状态。
```

## 2. 交叉引用：文章之间必须互相链接

### 规则

- **前置知识引用**：文章中首次出现一个非本文主题的重要概念（如"PPO"、"DDPM"）时，必须用超链接指向对应的前置知识文章
- **前置知识之间也要互相引用**：前置知识不是孤岛！如果 SAC 文章提到了 DDPG / TD3，就必须链接到对应的前置知识。如果对应的前置知识还没写，先留 `<!-- TODO: 需要前置知识文章：DDPG/TD3 -->` 占位，但**绝不能只一句话带过不加链接**。前置知识之间的网状引用关系是帮助读者构建知识体系的关键——读者应该能从任何一篇入口，顺着链接阅读完所有相关前置知识。
- **论文之间引用**：综述引用精读、精读引用综述、综述之间引用，只要内容相关就加链接
- **知识链接区**：每篇文章开头必须有"相关阅读"区域，列出读本文前建议了解的内容

### 链接格式

跨目录用绝对路径：
```markdown
如果对 PPO 不熟悉，请先阅读 [策略梯度与 PPO](/前置知识/000a_前置知识_策略梯度与PPO)。
```

同目录用相对路径：
```markdown
更多细节参见 [DPPO 精读](./001_DPPO_扩散策略策略优化)。
```

### 行文中的自然引用

不要只在开头放链接，**正文中提到相关概念时也要加链接**：

```markdown
DPPO 的核心思路是对扩散过程的每一步应用 [PPO 的 clip 机制](/前置知识/000a_前置知识_策略梯度与PPO#53-ppoproximal-policy-optimization-2017)……
```

## 3. 文章结构

每篇文章必须包含：

1. **Frontmatter**（title、order、tags、category）
2. **标题 + 元信息**（论文标题、一句话概括）
3. **相关阅读区**：前置知识 + 关联文章的链接列表
4. **贯穿全文的例子定义**：在引言部分定义一个具体场景，后面反复用
5. **正文**：先直觉 → 形式化 → 例子 → 代码/图示
6. **总结与对比表**
7. **延伸阅读**：相关文章的链接

---

- 行内公式：`$...$`

## 6. 表格优先于代码块

对比分析用 Markdown 表格。代码块只用于实际代码或伪代码。

---

## 7. 篇幅要求

| 类型 | 最低字数 | 说明 |
|------|----------|------|
| 前置知识 | 3000+ 字 | 一个概念讲透 |
| 论文精读 | 5000+ 字 | 逐段解读 |
| 综述 | 8000+ 字 | 系统覆盖，大量例子和对比 |

综述不是"列举方法名"，而是让读者读完后**真正理解**每个方法在做什么、为什么这么做、和其他方法有什么区别。

- 综述：`S0N_综述主题名.md`
