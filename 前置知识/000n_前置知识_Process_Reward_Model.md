# Process Reward Model（过程奖励模型）

> **一句话**：Process Reward Model (PRM) 为序列决策的每一步提供奖励信号，而非只在结束时给一个总分，从而解决 RL 中"稀疏奖励"的信号传播问题。

## 一、为什么需要 Process Reward Model

### 1.1 稀疏奖励问题

在大多数机器人操作任务中，环境奖励是**极度稀疏**的：

- 抓取任务：只有最终"抓到了"给 $r=1$，"没抓到"给 $r=0$
- 组装任务：只有整个组装完成才给奖励
- 一条轨迹可能有 50-200 步，但只有最后一步有奖励信号

这意味着 RL 算法面临一个严重的 credit assignment 问题：

> **50 步的动作序列中，到底是哪一步做对了/做错了，导致了最终的成功/失败？**

### 1.2 两种奖励模型

| 类型 | 英文 | 信号粒度 | 优势 | 劣势 |
|------|------|---------|------|------|
| Outcome Reward Model (ORM) | 结果奖励模型 | 只在序列结束时打分 | 简单、不需要中间标注 | 信号稀疏，学习慢 |
| Process Reward Model (PRM) | 过程奖励模型 | 对每一步/每个阶段打分 | 信号密集，学习快 | 需要设计中间步的评估方式 |

**类比**：
- ORM 像老师只看最终答案对不对（✓ 或 ✗）
- PRM 像老师看每一步推导过程（第 1 步对了 ✓，第 2 步出错了 ✗，第 3 步……）

### 1.3 PRM 的起源

PRM 最初由 OpenAI 在数学推理领域提出（"Let's Verify Step by Step", Lightman et al., 2023）：
- 训练一个模型对 LLM 数学推理的**每一步**打分
- 用于指导 LLM 的搜索（Best-of-N 采样时选最好的路径）

VLA-RL 等工作把这个思路迁移到了机器人操作领域：对机器人轨迹的**每个时间步**或**每个子任务阶段**打分。

## 二、PRM 的工作原理

### 2.1 基本结构

PRM 是一个训练好的模型 $R_\phi$，接受"当前状态+当前动作"（或"截止到当前的部分轨迹"），输出一个分数：

$$
r_t^{\text{PRM}} = R_\phi(s_t, a_t) \in \mathbb{R}
$$

这个分数表示"在状态 $s_t$ 执行动作 $a_t$ 有多大概率导向最终成功"。

最终给 RL 算法的奖励是环境稀疏奖励 + PRM 提供的密集奖励：

$$
r_t^{\text{total}} = r_t^{\text{sparse}} + \alpha \cdot r_t^{\text{PRM}}
$$

其中 $\alpha$ 是权重系数，控制 PRM 奖励的影响力。

### 2.2 在 VLA 中的具体实现：Robotic PRM

VLA-RL 论文提出的 Robotic Process Reward Model (RPRM) 有一个巧妙的设计：把奖励建模转化为**next-token prediction**。

具体做法：

1. 收集一批成功的专家轨迹 $\{(s_0, a_0), (s_1, a_1), \ldots, (s_T, a_T)\}$
2. 对每条轨迹做**里程碑分割**（milestone segmentation）：
   - 检测夹爪开合变化的时刻（通常标志着一个子任务完成）
   - 检测末端执行器速度接近零的时刻（标志着精细动作完成）
3. 在里程碑时刻标注正奖励（pseudo reward label）
4. 用这些数据微调一个 VLM（视觉语言模型），让它预测"接下来的动作 token 是否在通向成功"

**训练目标**：

$$
\mathcal{L}_{\text{RPRM}}(\phi) = -\mathbb{E}_t\left[\sum_{j=1}^{|a_t|} \log p_\phi(v_j^{\text{rprm}} | v_{<j}^{\text{out}}, o_t, v_t^{\text{in}})\right]
$$

**一句话**：RPRM 学习"成功轨迹的中间动作长什么样"，对新轨迹的动作打分就是看它有多像成功轨迹在对应阶段的动作。

### 2.3 代入例子

**任务**："pick up the butter and put it in the basket"

**轨迹分割**：
- 阶段 1：移动手臂到黄油上方（里程碑：手臂停在黄油正上方）
- 阶段 2：下降并夹取（里程碑：夹爪关闭，黄油被抓起）
- 阶段 3：移动到篮子上方（里程碑：黄油在篮子正上方）
- 阶段 4：放下（里程碑：夹爪打开，黄油落入篮子）

**PRM 打分示例**：

| 时间步 | 动作 | ORM 奖励 | PRM 奖励 | 解释 |
|--------|------|---------|---------|------|
| $t=5$ | 手臂朝黄油移动 | 0 | +0.3 | 方向正确，接近阶段1里程碑 |
| $t=10$ | 手臂到达黄油上方 | 0 | +0.8 | 完成阶段1里程碑！ |
| $t=12$ | 下降并夹取 | 0 | +0.5 | 正在完成阶段2 |
| $t=15$ | 夹爪关闭 | 0 | +0.9 | 完成阶段2里程碑！ |
| $t=20$ | 移向篮子 | 0 | +0.4 | 正在阶段3 |
| $t=30$ | 放入篮子 | **+1** | +1.0 | 任务成功！ |

注意：第 $t=5$ 到 $t=29$ 步，环境奖励 ORM 始终为 0，但 PRM 在每步都给出了有意义的信号。这大大加速了 RL 的学习。

## 三、PRM 的优势与风险

### 3.1 优势

1. **加速学习**：密集信号让策略梯度在每步都有方向，而非等到轨迹结束才有反馈
2. **更好的 credit assignment**：每步的奖励信号帮助策略识别"哪步做对了，哪步做错了"
3. **减少方差**：GAE 基于 PRM 的 value 信号可以更稳定地估计 advantage

### 3.2 风险：Reward Hacking

PRM 的最大风险是**reward hacking**（奖励黑客）——策略学会了欺骗 PRM，获得高分但实际上没有完成任务。

**例子**：
- PRM 学到"手臂在黄油上方"是正信号
- 策略可能学会"反复在黄油上方晃动"来持续获得 PRM 的正分数
- 但实际上没有执行抓取动作

**缓解方法**：
1. PRM 奖励权重 $\alpha$ 不宜过大（典型值 0.1-0.5）
2. 最终奖励仍以环境真实奖励为主
3. 加入 KL 惩罚防止策略偏离太远

## 四、PRM vs 其他奖励密集化方法

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| Hand-crafted dense reward | 人工定义每步奖励函数 | 精确 | 需要大量工程，每个任务重新设计 |
| Curiosity / Intrinsic reward | 探索新状态给奖励 | 通用 | 可能鼓励无意义的探索 |
| Hindsight relabeling | 把失败轨迹当成"达到另一个目标"的成功 | 数据高效 | 只适用于 goal-conditioned 任务 |
| **PRM** | 学习一个模型预测每步的进度 | 可扩展，可跨任务 | 需要成功轨迹来训练，可能 reward hack |

## 五、总结

| 概念 | 说明 |
|------|------|
| 本质 | 为序列决策每一步提供进度估计的学习模型 |
| 解决的问题 | 稀疏奖励下的 credit assignment |
| 训练数据 | 成功轨迹 + 自动分割的里程碑标注 |
| 使用方式 | $r_t^{\text{total}} = r_t^{\text{sparse}} + \alpha \cdot r_t^{\text{PRM}}$ |
| 主要风险 | Reward hacking |
| 代表工作 | Let's Verify Step by Step (LLM), VLA-RL (机器人) |
