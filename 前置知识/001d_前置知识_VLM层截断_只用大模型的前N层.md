# VLM 层截断：只用大模型的前 N 层

> 为什么可以把一个 28 层的大模型砍成 16 层来用？这么做不会丧失能力吗？本文从零解释"层截断"这个在 VLA 领域广泛使用的工程技术。

## 1. 什么是"层截断"？

### 1.1 直觉解释

想象一栋 28 层的大楼。每一层都有不同的功能：

- 1-5 层：物流和仓储（处理原始信息）
- 6-15 层：研发和设计（理解和推理）
- 16-25 层：市场和销售（把理解转化为文字）
- 26-28 层：顶层行政（最终决策输出）

现在有人告诉你："我不需要你写文章，我只需要你的理解能力。"
那你完全可以只用 1-16 层——跳过"把理解转化为文字"的那些楼层。

**层截断 (Layer Truncation)** 就是这个意思：只保留一个大型 Transformer 模型的前 N 层，
把后面的层**物理删除**，只使用前 N 层的输出作为特征。

### 1.2 严格定义

给定一个有 $L$ 层的 Transformer 模型 $f = f_L \circ f_{L-1} \circ \cdots \circ f_1$，
层截断是指只使用前 $N$ 层（$N < L$）：

$$
h_N = f_N \circ f_{N-1} \circ \cdots \circ f_1(x)
$$

其中 $h_N$ 是第 $N$ 层的输出，作为下游任务的输入特征。
层 $f_{N+1}, \ldots, f_L$ 被完全丢弃，不参与任何计算。

### 1.3 和"冻结"有什么区别？

| | 冻结 (Freeze) | 截断 (Truncate) |
|---|---|---|
| 参数是否存在？ | **是**，仍在内存中 | **否**，被物理删除 |
| 前向传播是否经过？ | **是**，仍要计算 | **否**，不计算 |
| 节省显存？ | 节省梯度显存 | 节省参数+激活值显存 |
| 节省计算？ | **不节省**（仍需前向） | **节省**（不需要计算） |
| 可恢复性？ | 可恢复（解冻即可） | 不可恢复（需重新加载） |

**关键区别**：冻结只是不让参数更新（不计算梯度），但前向传播仍然要跑完所有层。
截断是直接把层删掉——前向传播只跑到第 N 层就停了。

## 2. 为什么可以截断？——Transformer 的分层特征

### 2.1 不同层学到了什么？

大量研究表明，Transformer 模型的不同层有系统性的功能分化。
以一个 28 层的视觉-语言模型（VLM）为例：

```
层 1-4:    [底层] 局部特征
           - 边缘、纹理、颜色
           - 子词的基本表示
           
层 5-10:   [中低层] 模式识别
           - 物体部件、形状
           - 短语级语义
           
层 11-18:  [中层] 高级语义 ← 通常是最佳截断点
           - 完整物体识别
           - 空间关系理解
           - 跨模态对齐（图像区域↔文本描述）
           
层 19-24:  [中高层] 抽象推理
           - 逻辑推理链
           - 文本生成规划
           
层 25-28:  [顶层] 输出格式化
           - 语法正确性
           - 流畅性优化
           - 最终 token 预测
```

### 2.2 贯穿全文的例子

假设输入是一张桌面图片 + 指令 "pick up the red cube"。

**第 5 层输出**：模型知道"图中有红色区域、有方形区域"，但不确定它们是同一个物体。

**第 12 层输出**：模型知道"图中有一个红色方块，它在桌子中间偏左的位置"。

**第 20 层输出**：模型知道上面所有信息，另外还在规划"接下来要生成什么文字来描述这个场景"。

**第 28 层输出**：模型输出最终的文本 token logits："The red cube is on the left side of..."

对于机器人控制来说，**第 12 层的信息就够了**！我们需要的是"红色方块在哪里"，
而不是"如何用英语描述这个场景"。顶层的文本生成能力对机器人没有价值。

### 2.3 经验验证

多项研究（Probing 实验）证实了分层特征的存在：

| 任务 | 最佳层（28层模型） | 为什么 |
|------|---------|--------|
| 词性标注 | 4-6 | 底层就有足够的形态信息 |
| 命名实体识别 | 8-12 | 需要上下文来判断实体边界 |
| 语义相似度 | 12-18 | 需要理解含义 |
| 空间关系 | 14-20 | 需要跨模态推理 |
| 文本生成质量 | 24-28 | 需要顶层的语言模型能力 |

这就是为什么机器人 VLA 模型通常截断在 50-60% 的层数位置——
这正好是"语义理解最丰富、但还没开始做文本生成"的区间。

## 3. 怎么做截断？——代码实现

### 3.1 最简单的截断方式

```python
import torch
from transformers import AutoModel

# 1. 加载完整模型
model = AutoModel.from_pretrained("some-vlm-model")

# 2. 找到 Transformer 层的列表
# 不同模型的路径不同，需要 print(model) 确认
layers = model.language_model.layers  # 一个 nn.ModuleList

# 3. 物理删除多余的层
select_layer = 16  # 只保留前 16 层
while len(layers) > select_layer:
    layers.pop(-1)  # 从末尾逐个移除

# 4. 前向传播时用 output_hidden_states=True 取中间层输出
outputs = model(**inputs, output_hidden_states=True)
features = outputs.hidden_states[-1]  # 现在 [-1] 就是第 16 层
```

### 3.2 为什么用 `pop(-1)` 而不是切片？

你可能想写：
```python
model.language_model.layers = model.language_model.layers[:16]  # ❌ 不推荐
```

这样做有隐患——`nn.ModuleList` 被重新赋值后，PyTorch 的模块注册关系可能断裂
（`named_parameters()`、`state_dict()` 等方法可能出问题）。

用 `pop(-1)` 逐个移除是最安全的方式——它保持了 `nn.ModuleList` 的内部一致性。

### 3.3 如何确定层的路径？

不同的 VLM 架构，层列表的路径不同：

| 模型 | 层路径 |
|------|--------|
| Qwen3-VL | `model.language_model.layers` |
| InternVL / Eagle | `model.language_model.model.layers` |
| LLaMA 系列 | `model.model.layers` |
| GPT-2 | `model.transformer.h` |
| BERT | `model.encoder.layer` |

**找路径的方法**：
```python
print(model)  # 打印模型结构
# 找到类似 ModuleList 的部分，那就是 layers
```

## 4. 截断点如何选择？

### 4.1 通用规则

$$
N_{\text{optimal}} \approx 0.5L \sim 0.7L
$$

即保留总层数的 50%-70%。具体选择取决于下游任务对什么信息的需求：

| 下游任务 | 推荐截断比例 | 理由 |
|---------|------------|------|
| 图像分类 | 50-60% | 只需要物体识别级别的特征 |
| 视觉问答 | 60-70% | 需要跨模态对齐和推理 |
| 机器人控制 | 50-60% | 需要空间理解但不需要语言生成 |
| 文本摘要 | 90-100% | 需要完整的语言能力 |

### 4.2 GR00T 的选择：16/28 ≈ 57%

GR00T N1.7 对 Cosmos-Reason2-2B（约 28 层）截断到第 16 层。

为什么是 57%？
- 第 16 层包含了完整的视觉-语言对齐信息
- 再往上（17-28 层）主要用于生成文本——机器人不需要
- 57% 的截断节省了约 43% 的 LLM 计算量

### 4.3 如何验证截断点是否合适？

**方法 1：线性探测 (Linear Probing)**
- 在每一层的输出后面接一个简单的线性层
- 训练这个线性层做下游任务
- 哪一层的准确率最高，就截断到哪一层

**方法 2：直接实验**
- 尝试不同的 `select_layer` 值（如 8, 12, 16, 20, 24）
- 在目标任务上评估效果
- 选择效果和速度的最佳平衡点

## 5. 截断带来的好处与代价

### 5.1 好处

| 好处 | 具体数值（以 28→16 层为例） |
|------|--------------------------|
| 计算量减少 | LLM 部分减少 ~43%（节省 12/28 层的计算） |
| 显存减少 | 参数内存减少 ~43%，激活值内存也减少 |
| 推理延迟降低 | LLM 推理时间减少约 40% |
| 训练加速 | 即使冻结骨干，更短的前向传播也意味着更快的训练 |

**具体数值例子**：

假设 Qwen3-VL 2B 的 LLM 部分有 1.5B 参数（28层），每层约 54M 参数：
- 完整模型：28 层 × 54M = 1512M 参数，BF16 下占 ~3GB
- 截断到 16 层：16 × 54M = 864M 参数，BF16 下占 ~1.7GB
- **节省 1.3GB 显存**

推理延迟（假设每层 1ms）：
- 完整：28ms
- 截断：16ms
- **减少 12ms**（对 50Hz 控制来说很显著——一个控制周期只有 20ms）

### 5.2 代价

| 代价 | 说明 |
|------|------|
| 语言能力下降 | 失去文本生成能力（但机器人不需要） |
| 高级推理减弱 | 需要多步推理的任务（如数学计算）可能退化 |
| 不可逆 | 截断后不能恢复（但可以重新加载完整模型） |

### 5.3 为什么不直接用一个小模型？

一个合理的问题：既然要砍掉 12 层，为什么不直接训练一个只有 16 层的小模型？

答案是**预训练的价值**。一个 28 层模型在万亿 token 上预训练后：
- 前 16 层学到的特征质量远高于一个只有 16 层的模型从头训练
- 因为训练时梯度从第 28 层一路回传到第 1 层——底层受益于顶层的监督信号
- 截断后的前 16 层保留了"大模型训练"带来的特征质量

类比：一个读完博士（28层训练）的人，即使只让他做本科生的工作（截断到16层），
他的理解深度仍然远超一个只读到本科（16层从头训练）的人。

## 6. 截断 vs 其他特征提取方式

| 方式 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **层截断** | 物理删除后面的层 | 省显存+省计算 | 不可逆 |
| 中间层探针 | 保留完整模型，取中间层输出 | 可逆，灵活 | 不省计算 |
| 蒸馏 | 用大模型教小模型 | 模型小且完整 | 需要额外训练 |
| 量化 | 降低参数精度 | 保留所有层能力 | 精度可能下降 |

在 VLA 场景中，层截断是最直接有效的选择——因为：
1. 我们确实不需要顶层能力（不生成文本）
2. 推理延迟是硬约束（10-50Hz 控制频率）
3. 操作简单，不需要额外训练

## 7. 代码模板：通用层截断工具

如果你要在自己的项目中对任意 VLM 做层截断：

```python
def truncate_model_layers(model, layer_path: str, keep_layers: int):
    """
    通用层截断函数。
    
    Args:
        model: HuggingFace 模型
        layer_path: 层列表的属性路径 (如 "language_model.layers")
        keep_layers: 保留前多少层
    
    Example:
        truncate_model_layers(model, "language_model.layers", 16)
    """
    # 沿路径找到层列表
    parts = layer_path.split(".")
    obj = model
    for part in parts:
        obj = getattr(obj, part)
    
    # obj 现在是 nn.ModuleList
    original_count = len(obj)
    while len(obj) > keep_layers:
        obj.pop(-1)
    
    print(f"Truncated: {original_count} → {keep_layers} layers "
          f"(removed {original_count - keep_layers} layers)")
    
    return model
```

使用方式：
```python
from transformers import Qwen3VLForConditionalGeneration

model = Qwen3VLForConditionalGeneration.from_pretrained("nvidia/Cosmos-Reason2-2B")
model = truncate_model_layers(model, "language_model.layers", keep_layers=16)

# 前向传播
outputs = model(**inputs, output_hidden_states=True)
features = outputs.hidden_states[-1]  # 第16层的输出 [B, seq_len, hidden_dim]
```

## 8. 总结

| 要点 | 内容 |
|------|------|
| **是什么** | 物理删除 Transformer 的后面几层，只保留前 N 层 |
| **为什么能这样做** | 不同层有不同功能，下游任务通常只需要中间层的语义特征 |
| **什么时候用** | 当你不需要模型的"生成能力"，只需要"理解能力"时 |
| **截断到哪里** | 通常 50-70% 的位置（具体需要实验验证） |
| **工程实现** | `while len(layers) > N: layers.pop(-1)` |
| **和冻结的区别** | 截断连前向传播都不做，冻结只是不更新参数 |

层截断是 VLA 领域的标准工程实践。GR00T、π₀ 等模型都使用这个技术来平衡
视觉-语言理解能力和推理延迟——在不损失机器人控制所需信息的前提下，
尽可能减少骨干网络的计算开销。
