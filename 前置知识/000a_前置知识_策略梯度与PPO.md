## 第一部分：强化学习的基本元素

### 1.1 Agent 和 Environment 的交互循环

强化学习的核心场景非常简单：一个"智能体（agent）"和一个"环境（environment）"来回交互。

$$
循环过程:
  时刻 t=0: 环境给出初始状态 s₀
  时刻 t=0: agent 根据策略选择动作 a₀
  时刻 t=0: 环境给出奖励 r₀，并转移到新状态 s₁
  时刻 t=1: agent 看到 s₁，选择 a₁
  时刻 t=1: 环境给出 r₁，转移到 s₂
  ...
  直到 episode 结束
$$

### 1.2 每个元素的含义

**状态 s（State）**：agent 能观测到的环境信息。

```text
机器人例子:
  s = [关节角度7维, 关节速度7维, 末端位姿6维, 目标位置3维] = 23维向量

游戏例子:
  s = 屏幕像素 84×84×3

LLM 例子:
  s = 当前已生成的所有 token 序列
```

**动作 a（Action）**：agent 对环境施加的操作。

$$
机器人例子:
  a = [7个关节力矩] 或 [末端位移 Δx, Δy, Δz, Δroll, Δpitch, Δyaw]
  → 连续动作空间（每个维度是实数）

游戏例子:
  a ∈ {上, 下, 左, 右, 跳, 攻击}
  → 离散动作空间

LLM 例子:
  a = 下一个生成的 token（从词表中选一个）
  → 离散动作空间（词表大小，通常 32000+）
$$


**奖励 r（Reward）**：一个标量数字，告诉 agent "这一步做得好不好"。

```text
机器人例子:
  稀疏奖励: 任务完成=1, 失败=0（只在最后一步给信号）
  密集奖励: 每步给 -距离目标的距离 + 完成后给大 bonus

游戏例子:
  吃到金币 +10, 碰到怪物 -50, 通关 +1000

LLM 例子:
  人类偏好奖励模型给的分数（RLHF）
```

**策略 π（Policy）**：agent 的"决策规则"。给定当前状态，输出一个动作（或动作的概率分布）。

$$
确定性策略:  a = μ(s)                    给状态，直接输出一个动作
随机策略:    a ~ π(a|s)                  给状态，输出动作的概率分布，然后从中采样
参数化策略:  a ~ π_θ(a|s)               用神经网络（参数 θ）表示策略
$$

**在深度 RL 中，策略就是一个神经网络。** 输入是 s，输出是 a 的分布参数（比如均值和方差）。

### 1.3 目标：最大化累积折扣奖励

一个 episode（一整轮交互）产生一条轨迹：

$$
τ = (s₀, a₀, r₀, s₁, a₁, r₁, ..., s_T, a_T, r_T)
$$

我们的目标是最大化**累积折扣奖励**：

$$
R(τ) = r₀ + γ·r₁ + γ²·r₂ + ... + γᵀ·r_T = Σ_{t=0}^{T} γᵗ × r_t
$$

**符号含义**：
- γ ∈ (0, 1)：折扣因子（discount factor）
- γ = 0.99：几乎不折扣，很重视长期奖励
- γ = 0.9：较快衰减，更重视近期奖励
- γ 的直觉：未来的奖励因为有不确定性，所以"打个折"

**为什么要折扣？**
1. 数学上保证无限 horizon 时 R(τ) 有界
2. 直觉上：一块钱今天给你比明天给你更有价值（时间偏好）
3. 实际中：越远的未来越不可控，给更低权重合理

### 1.4 策略的性能指标 J(θ)

$$
J(θ) = E_{τ~π_θ} [R(τ)] = E_{τ~π_θ} [Σ_{t=0}^{T} γᵗ × r_t]
$$

**含义**：如果用策略 π_θ 和环境交互很多次，平均每次能拿多少累积奖励。

**强化学习的目标就是找到 θ*，使得 J(θ*) 最大。**

## 第二部分：Value Function 和 Advantage Function

### 2.1 状态价值函数 V(s)

$$
V^π(s) = E_{τ~π} [Σ_{t'=t}^{T} γ^{t'-t} × r_{t'} | s_t = s]
$$

**白话**：从状态 s 开始，按照策略 π 一直走下去，平均能拿到多少未来总奖励。

$$
例子:
  围棋中: V(当前盘面) = 在这个局面下，我方的胜率
  机器人: V(当前关节状态) = 从现在开始能完成任务的期望总奖励
$$

V(s) 高 → 这个状态"前途光明"
V(s) 低 → 这个状态"前途暗淡"

### 2.2 动作价值函数 Q(s, a)

$$
Q^π(s, a) = E_{τ~π} [Σ_{t'=t}^{T} γ^{t'-t} × r_{t'} | s_t = s, a_t = a]
$$

**白话**：在状态 s 做了动作 a 之后，再按策略 π 走下去，平均能拿多少。

Q 和 V 的关系：

$$
V(s) = E_{a~π(·|s)} [Q(s, a)] = Σ_a π(a|s) × Q(s, a)
$$

即 V(s) 是对所有动作的 Q 值按策略概率加权平均。

### 2.3 优势函数 A(s, a)——策略梯度的核心

$$
A^π(s, a) = Q^π(s, a) - V^π(s)
$$

**白话**：在状态 s 做动作 a，比"按策略平均水平做"好多少（或差多少）。

$$
A(s, a) > 0: 这个动作比平均好 → 应该增加它的概率
A(s, a) < 0: 这个动作比平均差 → 应该减少它的概率
A(s, a) = 0: 这个动作就是平均水平 → 概率不变
$$

**为什么 Advantage 比 Q 或 R 更好？**

如果直接用累积奖励 R_t 做策略梯度：
- 所有动作的梯度方向都是正的（因为 R_t 通常大于 0）
- 只是"好的动作正得多，差的动作正得少"
- 方差很大，收敛慢

用 Advantage：
- 好动作 → 正梯度（增加概率）
- 差动作 → 负梯度（减少概率）
- 平均动作 → 零梯度（不变）
- 方差大幅降低

## 第三部分：策略梯度定理（Policy Gradient Theorem）

### 3.1 问题：怎么对 J(θ) 求梯度？

我们想计算 ∇_θ J(θ) 来做梯度上升（最大化 J）。但 J(θ) 包含了对整条轨迹的期望，轨迹又依赖于环境的转移概率 P(s'|s,a)——我们不知道 P 的解析形式。

看起来似乎没法求梯度？

### 3.2 策略梯度定理的推导

**核心 trick：Log-Derivative Trick（对数导数技巧）**

对任何概率分布 p_θ(x)：

$$
∇_θ p_θ(x) = p_θ(x) × ∇_θ log p_θ(x)
$$

证明很简单：∇_θ log p_θ(x) = ∇_θ p_θ(x) / p_θ(x)，两边乘以 p_θ(x) 就得到上式。

**用这个 trick 推导策略梯度**：

$$
∇_θ J(θ) = ∇_θ E_{τ~π_θ}[R(τ)]
          = ∇_θ ∫ p_θ(τ) × R(τ) dτ        ← 期望展开为积分
          = ∫ ∇_θ p_θ(τ) × R(τ) dτ         ← 梯度推进积分内
          = ∫ p_θ(τ) × ∇_θ log p_θ(τ) × R(τ) dτ   ← log-derivative trick
          = E_{τ~π_θ}[∇_θ log p_θ(τ) × R(τ)]        ← 变回期望形式
$$

**现在展开 log p_θ(τ)**：

一条轨迹的概率是：

$$
p_θ(τ) = p(s₀) × ∏_{t=0}^T [π_θ(a_t|s_t) × P(s_{t+1}|s_t, a_t)]
$$

取对数：

$$
log p_θ(τ) = log p(s₀) + Σ_t [log π_θ(a_t|s_t) + log P(s_{t+1}|s_t, a_t)]
$$

对 θ 求梯度时，p(s₀) 和 P(s'|s,a) 都和 θ 无关，梯度为 0！只剩策略项：

$$
∇_θ log p_θ(τ) = Σ_{t=0}^T ∇_θ log π_θ(a_t|s_t)
$$

### 3.3 策略梯度最终形式

$$
∇_θ J(θ) = E_{τ~π_θ} [Σ_{t=0}^T ∇_θ log π_θ(a_t|s_t) × R_t]
$$

其中 R_t = Σ_{t'=t}^T γ^{t'-t} × r_{t'} 是从 t 时刻开始的折扣累积奖励。

**为什么只用 R_t 而不是 R(τ)?** 因为 t 时刻之前的奖励和 a_t 的选择无关。只有未来奖励才能作为 a_t 的反馈信号。这个优化叫 "reward-to-go"。

### 3.4 REINFORCE 算法

最简单的策略梯度算法：

$$
REINFORCE 算法:
1. 用当前策略 π_θ 跑 N 条轨迹 τ₁, τ₂, ..., τ_N
2. 对每条轨迹的每个 (s_t, a_t)，计算 R_t = Σ_{t'≥t} γ^{t'-t} r_{t'}
3. 用蒙特卡罗近似梯度:
   ∇_θ J ≈ (1/N) × Σ_i Σ_t ∇_θ log π_θ(a_t^i | s_t^i) × R_t^i
4. θ ← θ + α × ∇_θ J
$$

**REINFORCE 的问题**：

```text
方差极大！
因为 R_t 是随机变量（取决于后续所有随机转移和动作选择），
不同轨迹的 R_t 差异可能非常大。

结果：需要海量的轨迹才能得到可靠的梯度估计。
     收敛极慢。实际中几乎不直接用。
```

### 3.5 引入 Baseline 降低方差

可以证明：对策略梯度加一个和 a 无关的 baseline b(s) 不改变期望（无偏），但能降低方差：

$$
∇_θ J(θ) = E[∇_θ log π_θ(a|s) × (R_t - b(s_t))]
$$

最优 baseline 接近 V(s_t)。所以用 A(s,a) = Q(s,a) - V(s) 替换 R_t - b(s)：

$$
∇_θ J(θ) = E[∇_θ log π_θ(a_t|s_t) × A^π(s_t, a_t)]
$$

这就是现代策略梯度算法的基本形式。

## 第四部分：GAE（广义优势估计）

### 4.1 估计 Advantage 的挑战

A(s,a) = Q(s,a) - V(s) 是理论量，实际中怎么估计？

**方法 1：Monte Carlo（用真实回报）**

$$
A_t^MC = R_t - V(s_t) = (Σ_{t'≥t} γ^{t'-t} r_{t'}) - V(s_t)
$$

- 无偏估计（期望等于真实 A）
- 方差很大（因为 R_t 随机性大）

**方法 2：TD（时序差分，只看一步）**

$$
δ_t = r_t + γ × V(s_{t+1}) - V(s_t)
A_t^TD = δ_t
$$

- 有偏（因为 V(s_{t+1}) 本身是估计值，不精确）
- 方差小（只涉及一步随机性）

**核心矛盾**：一步 TD 偏差大但方差小；多步/完整回报偏差小但方差大。

### 4.2 GAE 的思路：加权混合所有步数的估计

先定义不同步数的 advantage 估计：

$$
A_t^(1) = δ_t = r_t + γV(s_{t+1}) - V(s_t)                    ← 1步
A_t^(2) = δ_t + γδ_{t+1} = r_t + γr_{t+1} + γ²V(s_{t+2}) - V(s_t)  ← 2步
A_t^(3) = δ_t + γδ_{t+1} + γ²δ_{t+2}                          ← 3步
...
A_t^(∞) = Σ_{l=0}^{∞} γ^l δ_{t+l} = R_t - V(s_t)             ← 完整 MC
$$

GAE 对这些估计做指数加权平均：

$$
A_t^GAE(γ,λ) = (1-λ) × [A_t^(1) + λA_t^(2) + λ²A_t^(3) + ...]
             = Σ_{l=0}^{∞} (γλ)^l × δ_{t+l}
$$

**λ 的作用**：

$$
λ = 0: A_t^GAE = δ_t            ← 纯 TD，低方差高偏差
λ = 1: A_t^GAE = R_t - V(s_t)   ← 纯 MC，低偏差高方差
λ = 0.95: 适中的权衡            ← 实践中最常用
$$

### 4.3 GAE 的计算实现

从后往前递归计算，非常高效：

$$
A_T = δ_T                                    ← 最后一步
A_{T-1} = δ_{T-1} + γλ × A_T                 ← 倒数第二步
A_{T-2} = δ_{T-2} + γλ × A_{T-1}             ← 继续往前
...
A_t = δ_t + γλ × A_{t+1}                     ← 通用递推公式
$$

```python
# 伪代码
advantages = zeros(T)
last_adv = 0
for t in reversed(range(T)):
    delta = rewards[t] + gamma * values[t+1] * (1 - dones[t]) - values[t]
    advantages[t] = delta + gamma * lam * (1 - dones[t]) * last_adv
    last_adv = advantages[t]
```

这就是所有 PPO 实现里都能看到的 GAE 计算循环。

## 第五部分：从 REINFORCE 到 PPO——解决训练不稳定问题

### 5.1 策略梯度的灾难性问题

即使有了 GAE 降低方差，直接用策略梯度更新仍然有一个严重问题：

**步长选择极度困难。**

```text
学习率太大:
  一次更新改变策略太多 → 新策略性能可能骤降（掉下悬崖）
  新策略产生的轨迹和旧策略完全不同 → 之前的 advantage 估计不再适用
  → 连环崩溃

学习率太小:
  收敛极慢
  在复杂任务上可能永远学不到有用的东西
```

**直觉类比**：你在山上蒙着眼走，只能靠脚下的坡度（梯度）判断方向。如果步子太大，可能一脚踏入悬崖。如果步子太小，天黑前走不到山顶。

### 5.2 信任域方法（Trust Region）的思路

核心想法：**限制每次更新后，新策略和旧策略不能差太多。**

"差多少"用 KL 散度衡量：

$$
KL(π_old ‖ π_new) = E_{s~d^π_old} [KL(π_old(·|s) ‖ π_new(·|s))]
$$

**TRPO（Trust Region Policy Optimization, 2015）**：

$$
max_θ  E_{s,a ~ π_old} [π_θ(a|s) / π_old(a|s) × A^π_old(s,a)]

subject to:  KL(π_old ‖ π_θ) ≤ δ
$$

翻译：在"新旧策略 KL 散度不超过 δ"的约束下，最大化性能改进。

**TRPO 的问题**：
- 需要用共轭梯度法求解 KL 约束下的最优步长
- 需要计算 Fisher 信息矩阵和向量积
- 实现复杂，调参困难
- 不好和大规模并行训练配合

### 5.3 PPO：用 Clip 代替 KL 约束

PPO 的天才之处：**用一个简单的 clip 操作代替复杂的 KL 约束**，达到几乎一样的效果。

定义**概率比（probability ratio）**：

$$
r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)
$$

含义：
- r_t = 1 → 新旧策略对这个 (s,a) 对给出相同概率
- r_t = 1.5 → 新策略比旧策略更倾向选这个动作（概率高了 50%）
- r_t = 0.5 → 新策略比旧策略更不倾向选这个动作（概率低了 50%）

**PPO-Clip 目标函数**：

$$
L_CLIP(θ) = E_t [ min(
    r_t(θ) × Â_t,
    clip(r_t(θ), 1-ε, 1+ε) × Â_t
)]
$$

**ε 是 clip 范围**，通常 0.1 或 0.2。

### 5.4 Clip 机制的详细解析

**情况 1：Â_t > 0（这个动作比平均好）**

我们想增加 r_t（增加好动作的概率）。

$$
min(r_t × Â_t, clip(r_t, 1-ε, 1+ε) × Â_t)

当 r_t < 1+ε: 取 r_t × Â_t（正常梯度，鼓励增大概率）
当 r_t > 1+ε: 取 (1+ε) × Â_t（封顶了！不让概率继续增大）
$$

→ 好动作的概率只能增加到原来的 (1+ε) 倍，不能更多。防止"一下子把所有概率都压到一个动作上"。

**情况 2：Â_t < 0（这个动作比平均差）**

我们想减小 r_t（减少差动作的概率）。

$$
min(r_t × Â_t, clip(r_t, 1-ε, 1+ε) × Â_t)

注意 Â_t < 0，所以 r_t × Â_t 越大（越不负）越好
→ 想让 r_t 变小（减少概率）

当 r_t > 1-ε: 取 r_t × Â_t（正常梯度，鼓励减小概率）
当 r_t < 1-ε: 取 (1-ε) × Â_t（封底了！不让概率继续减小）
$$

→ 差动作的概率只能减少到原来的 (1-ε) 倍，不能更少。防止"把一个动作的概率压到 0"。

### 5.5 为什么 Clip 能防止策略崩溃

$$
没有 Clip（普通策略梯度）:
  迭代 1: 发现动作 a* 在某个状态 s* 下 Advantage 很高
  迭代 2: 疯狂增加 π(a*|s*) → 比如从 0.3 增到 0.99
  迭代 3: 策略几乎变成确定性的 → 探索消失 → 无法发现更好的策略
  → 可能陷入局部最优或崩溃

有 Clip (ε=0.2):
  迭代 1: 发现 a* 好，r = π_new/π_old 增大
  迭代 2: r 最多到 1.2（被 clip 住了）→ 概率只增加 20%
  迭代 3: 基于新策略重新采样 → 重新评估 Advantage → 如果确实好就继续增加
  → 缓慢、平稳地改进策略，不会一下子崩
$$

## 第六部分：PPO 的完整训练流程

### 6.1 一个 PPO 训练 iteration 的完整步骤

$$
for iteration = 1, 2, 3, ...:

  ═══════════════ 阶段 1：数据采集 ═══════════════
  
  用当前策略 π_θ_old 在 N 个并行环境中各跑 T 步
  → 得到 N×T 个 (s, a, r, s', done) 样本
  → 记录每个 (s,a) 对应的 log π_θ_old(a|s)（旧策略的 log 概率）
  
  ═══════════════ 阶段 2：Advantage 计算 ═══════════════
  
  用 Critic 网络 V_φ(s) 估计每个状态的 value
  用 GAE 计算每个 (s,a) 的 advantage:
    δ_t = r_t + γ × V(s_{t+1}) × (1-done_t) - V(s_t)
    A_t = Σ_{l=0}^{...} (γλ)^l × δ_{t+l}   (递推实现)
  
  计算 reward-to-go（用于 Critic 的训练目标）:
    R_t^target = A_t + V(s_t)
  
  对 advantage 做归一化:
    A_t ← (A_t - mean(A)) / (std(A) + 1e-8)
  
  ═══════════════ 阶段 3：策略和 Critic 更新 ═══════════════
  
  for epoch = 1, ..., K:  (通常 K=3~10)
    
    对收集的数据做 mini-batch 随机打乱
    
    for each mini-batch:
      
      # Actor (策略) 更新
      计算新策略下的 log π_θ(a|s)
      r_t(θ) = exp(log π_θ(a|s) - log π_θ_old(a|s))  # 概率比
      L_clip = min(r_t × A_t, clip(r_t, 1-ε, 1+ε) × A_t)
      L_actor = -mean(L_clip)  # 取负号因为是最小化 loss
      
      # Critic (价值函数) 更新
      V_pred = V_φ(s)
      L_critic = MSE(V_pred, R_t^target)
      
      # 可选: Entropy bonus（鼓励探索）
      L_entropy = -mean(entropy(π_θ(·|s)))
      
      # 总 loss
      L_total = L_actor + c₁ × L_critic + c₂ × L_entropy
      
      反向传播，更新 θ 和 φ
    
    # 可选：如果 KL 散度超过阈值，提前终止这个 iteration 的更新
    if mean_KL > KL_threshold:
      break
  
  ═══════════════ 阶段 4：更新旧策略 ═══════════════
  θ_old ← θ
$$

### 6.2 为什么要做多个 epoch

传统策略梯度（REINFORCE, A2C）：**数据只用一次就扔掉。** 因为策略变了以后，旧数据的梯度方向就不对了（on-policy 约束）。

PPO 的 clip 机制允许**同一批数据反复使用多次**（通常 3-10 次），因为 clip 限制了策略变化幅度。只要策略还没变太多（r_t 还在 [1-ε, 1+ε] 内），旧数据的梯度方向大致还是对的。

这大幅提高了样本效率——不用每算一次梯度就重新采集数据。

### 6.3 Advantage 归一化

```text
A_t ← (A_t - mean(A)) / (std(A) + 1e-8)
```

为什么要这样做？

1. 不同任务的奖励量级不同。归一化后 Advantage 均值为 0、标准差为 1，使得 clip ratio ε 的意义在不同任务间可比。
2. 梯度大小更稳定。不会因为某些 episode 奖励特别高就产生特别大的梯度。
3. 对超参数（学习率、clip ratio）更鲁棒。

## 第七部分：连续动作空间中的策略梯度

### 7.1 高斯策略

机器人控制中最常用的策略参数化：

$$
π_θ(a|s) = N(a; μ_θ(s), σ²)

其中:
  μ_θ(s): 神经网络输出的均值向量（和 s 有关）
  σ: 标准差（可以是固定的，也可以是学习的）
  a: 从这个高斯中采样的动作
$$

**对数概率的计算（PPO 需要这个）**：

$$
log π_θ(a|s) = -1/(2σ²) × ‖a - μ_θ(s)‖² - d/2 × log(2πσ²)

其中 d 是动作维度
$$

这个公式告诉我们：动作 a 离均值 μ_θ(s) 越远，log 概率越负（概率越小）。

**概率比的计算**：

$$
r(θ) = π_θ(a|s) / π_θ_old(a|s) 
     = exp(log π_θ - log π_θ_old)
     = exp(-1/(2σ²) × (‖a-μ_new‖² - ‖a-μ_old‖²))
$$

### 7.2 为什么高斯策略有局限

**单峰问题**：高斯只有一个均值。如果真实最优动作有多种模式（比如"左绕"和"右绕"都行），高斯策略会在两个模式之间取平均，输出一个两边都不是的动作。

$$
真实最优动作: 要么 a=-1（左绕），要么 a=+1（右绕）
高斯策略学到: μ=0, σ=1 → 平均值是中间 → 直接撞墙
$$

这就是为什么 Diffusion Policy 要用扩散模型代替高斯——扩散模型可以表示任意复杂的多模态分布。

### 7.3 在 DPPO 中：每步去噪的高斯

DPPO 把扩散策略的每一步去噪写成高斯：

$$
p_θ(a_t^k | a_t^{k+1}, s_t) = N(a_t^k; μ_θ(a_t^{k+1}, k+1, s_t), σ_{k+1}²I)
$$

所以每一步的 log 概率就是标准高斯 log-likelihood。PPO 的 clip 可以直接对每一步去噪计算概率比 r(θ)。

**整条去噪链的概率**：

```text
如果要算整体 π(a_0|s) 需要对中间变量 a_1,...,a_K 积分（不可解析）
但 DPPO 不需要整体概率！它对每一步单独做 PPO 更新。
```

## 第八部分：PPO 在实际中的关键超参数

### 8.1 超参数表与调参经验

| 参数 | 符号 | 典型范围 | 效果 |
|------|------|---------|------|
| 折扣因子 | γ | 0.99-0.999 | 越大越重视长期奖励 |
| GAE λ | λ | 0.9-0.99 | 越大越低偏差高方差 |
| Clip ratio | ε | 0.1-0.3 | 越大允许更大策略变化 |
| PPO epochs | K | 3-10 | 每批数据更新几轮 |
| Mini-batch size | - | 256-8192 | 梯度估计的精度 |
| 学习率 | α | 1e-5 ~ 3e-4 | 太大不稳定，太小收敛慢 |
| 并行环境数 | N | 8-10000+ | 越多数据越多 |
| Entropy coeff | c₂ | 0.0-0.01 | 越大越鼓励探索 |
| Value loss coeff | c₁ | 0.5-1.0 | Critic 训练权重 |

### 8.2 调参直觉

**Clip ratio ε**：
- DPPO 论文中发现：操作任务用 ε=0.01（非常保守），运动任务用 ε=0.1
- 判断标准：看 "clipping fraction"——被 clip 住的样本比例。理想是 10-20%。如果太高说明更新太激进。

**γ 的选择**：
- γ=0.99：episode 约 100 步时合适（有效视野 ~100 步）
- γ=0.999：episode 约 1000 步时用（有效视野 ~1000 步）
- AGILE 论文中机器人操作任务统一用 γ=0.999

**并行环境数 N**：
- GPU 并行（Isaac Gym）：通常 1000-10000 个环境
- CPU 并行（MuJoCo）：通常 40-100 个环境
- 更多并行 → 每个 iteration 数据更多 → advantage 估计更准 → 收敛更快

### 8.3 PPO 的常见失败模式

```text
失败模式 1: 奖励不涨
  可能原因: 学习率太小 / 探索不足（entropy coeff 太小）/ 奖励信号太稀疏
  诊断: 看 entropy 是否在降但奖励没涨

失败模式 2: 奖励先涨后崩
  可能原因: clip ratio 太大 / 训练 epoch 太多 / value function 不准
  诊断: 看 KL 散度是否突然变大

失败模式 3: 性能波动大（不同种子差异大）
  可能原因: 奖励设计有多个局部最优 / 初始化敏感
  诊断: 跑 5+ 个种子看方差

失败模式 4: Value loss 居高不下
  可能原因: 奖励量级太大 / 奖励在 curriculum 中剧变
  诊断: 用 reward normalization
```

## 第九部分：PPO 为什么是"万金油"——适用范围

### 9.1 PPO 在不同领域的应用

```text
游戏 AI:
  - OpenAI Five (Dota 2): PPO 训练，5v5 击败职业选手
  - AlphaStar (StarCraft): PPO 作为核心 RL 算法之一

LLM 对齐:
  - InstructGPT/ChatGPT: RLHF 中的 RL 部分就是 PPO
  - 对 LLM 输出的 token 序列做策略梯度

机器人运动:
  - Isaac Gym 中的四足/双足运动: 几千并行环境 + PPO，几小时训好
  - AGILE 人形机器人: PPO + 各种正则化

机器人操作:
  - DPPO: 对扩散策略的每步去噪做 PPO
  - FurnitureBench: PPO 微调长 horizon 操作
```

### 9.2 PPO 的核心优势

1. **On-policy + 多 epoch 的折中**：比纯 on-policy（A2C）样本效率高，比 off-policy（[SAC](/前置知识/000k_前置知识_SAC_Soft_Actor_Critic), [TD3](/前置知识/000q_前置知识_TD3)）稳定
2. **并行友好**：天然适配大规模 GPU 并行仿真
3. **实现简单**：核心代码 <200 行（参见 CleanRL）
4. **通用性**：连续/离散动作，状态/像素输入都行
5. **调参相对友好**：超参数之间耦合少，各管各的

### 9.3 PPO 的局限

1. **样本效率不如 off-policy 方法**：每个 iteration 的数据用完就扔（虽然用了多 epoch，但比 replay buffer 还是少）
2. **需要大量并行环境**：如果只有一个真实机器人，PPO 很难用
3. **on-policy 特性**：不能利用历史数据。之前跑过的轨迹不能重用。
4. **信用分配困难**：对于很长 horizon（几千步）的任务，GAE 的 advantage 估计也可能不准

## 第十部分：从 PPO 到 DPPO 的桥梁

### 10.1 标准 PPO 的关键操作

$$
1. 采样动作:     a ~ π_θ(a|s)
2. 算 log 概率:  log π_θ(a|s)
3. 算概率比:     r(θ) = π_θ(a|s) / π_old(a|s)
4. Clip 更新:    min(r×A, clip(r, 1-ε, 1+ε)×A)
$$

### 10.2 DPPO 把这些操作映射到去噪步

$$
1. 采样动作 → 跑完整的去噪链 (a_K → a_{K-1} → ... → a_0)
2. log 概率 → 每步去噪的 log N(a_{k-1}; μ_θ(a_k, k, s), σ_k²)
3. 概率比   → 对每步去噪分别算 r_k(θ) = p_θ_new(a_{k-1}|a_k,s) / p_θ_old(a_{k-1}|a_k,s)
4. Clip    → 对每步去噪分别做 clip（而且可以对不同 k 用不同的 ε！）
$$

### 10.3 为什么这是合法的

策略梯度定理只要求：
1. 能算 log π(a|s)（或其无偏估计）
2. 能用策略采样轨迹

DPPO 把去噪展开成 MDP 后，每步去噪就是一个"动作"，有显式的高斯 log-likelihood。策略梯度定理对这个展开的 MDP 完全适用。
