# 前置知识：Behler-Parrinello 局部能量框架——为什么机器学习力场可以"线性扩展"

> **一句话**：把一个大系统的总能量拆成"每个原子/格点各自贡献一份"，且每一份只依赖它周围的局部环境——这样无论系统有多大，训练一次的模型都能直接套用，计算量随系统尺寸线性增长而不是爆炸式增长。

**前置概念**：基础的分子动力学/统计物理直觉（"系统总能量"、"力是能量的导数"）；无需先学神经网络细节

---

## 贯穿全文的例子

> 假设你要预测一栋 100 层大楼每层楼的重量分布。如果你的方法是"必须同时看完所有 100 层的图纸才能算出任意一层的重量"，那么当大楼变成 200 层时，你的方法完全失效，需要重新设计。但如果你的方法是"每层楼的重量只取决于这层楼上下相邻的 2 层楼的结构"，那么无论大楼有 100 层还是 10000 层，同一套方法都能直接套用，而且计算量只是线性增加。

这个"只看局部邻居就能推算局部性质"的思想，正是 Behler-Parrinello（BP）框架的核心，也是本文将要介绍的等变神经网络力场能够做到"大规模可扩展"的根本原因。

---

## 一、问题的起点：为什么直接算能量代价高昂

### 1.1 量子力学计算的瓶颈

在凝聚态物理和材料模拟中，一个系统（比如晶格上耦合的电子）的能量严格来说需要求解电子的哈密顿量 $\hat H_e$：

$$
E = \langle \hat H_e \rangle = \mathrm{Tr}\big[\hat\varrho_e \hat H_e(\{\Phi_i\})\big]
$$

**为什么需要这个公式**：我们做动力学模拟时,每一步都要知道系统当前构型下的能量（以及由能量算出的力），才能推演下一步怎么运动。这个式子给出了"能量"的严格定义。

> **一句话直觉**：能量就是把电子的量子态在哈密顿量下做"加权平均"。

**逐项拆解**：

| 符号 | 含义 |
|------|------|
| $\hat H_e(\{\Phi_i\})$ | 电子哈密顿量，依赖于晶格每个格点 $i$ 上的经典变量 $\Phi_i$（比如局部晶格畸变、局部磁矩） |
| $\hat\varrho_e = e^{-\beta \hat H_e}/Z_e$ | 多电子密度算符（热平衡态） |
| $Z_e$ | 电子系统的配分函数 |
| $\mathrm{Tr}[\cdot]$ | 对所有电子量子态求"迹"，等价于求期望 |

**问题所在**：计算 $\hat\varrho_e$（本质上是对角化整个电子哈密顿量）的计算代价，随系统格点数 $N$ 呈**超线性**增长（对角化一个 $N\times N$ 矩阵的标准代价是 $O(N^3)$）。而动力学模拟往往需要在每一个时间步都重新做这个对角化——这构成了大规模动力学模拟中最主要的计算瓶颈。

### 1.2 一个"看似合理但行不通"的思路

既然神经网络号称"万能函数逼近器"，一个自然的想法是：直接训练一个网络，输入整个系统的全部构型 $\{\Phi_i\}$，输出全部格点的力 $\{F_i\}$。

但这个思路有两个致命问题：

1. **不能迁移到不同尺寸的系统**：在 $10\times 10$ 格点上训练出的网络，输入维度是固定的，没法直接用到 $100\times 100$ 的格点上——输入输出的形状都不匹配。
2. **计算代价仍随系统尺寸超线性增长**：一个"看全局"的网络，其内部计算量通常也会随输入规模变差得很快，没有从根本上解决问题。

## 二、近视性原理：物理学给出的救命稻草

### 2.1 W. Kohn 的洞见

凝聚态物理学家沃尔特·科恩（W. Kohn，1998 年诺贝尔化学奖获得者）指出：无论是绝缘体还是金属，很多电子性质都主要取决于**局部环境**，而不是整个系统的全貌。这个现象被称为**近视性原理**（nearsightedness principle）——电子仿佛"近视眼"一样，看不清远处发生的事，只对身边的环境敏感。

物理根源在于波动力学中的**相消干涉**（destructive interference）：来自远处格点的电子波函数贡献，经过叠加后大部分会互相抵消，只留下近处环境的净效应。

> **一句话直觉**：一个格点受到的"净影响"主要来自它周围一圈邻居，远处发生的事对它几乎没有累积效应。

### 2.2 近视性原理如何拯救了机器学习力场

如果每个格点的物理量确实只依赖有限范围内的邻居，那么我们完全没必要训练一个"看全局"的网络——只需要训练一个**只看局部邻域**的小网络，然后把这同一个小网络**平移复制**应用到系统里的每一个格点上。这样一来：

- 训练好的模型天然可以迁移到任意大小的系统（只要局部环境的统计规律相似）；
- 每个格点的计算代价是常数（只处理固定大小的局部邻域），总计算量随格点数 $N$ **线性**增长，这就是"linear-scaling"（线性扩展）力场方法的由来。

## 三、Behler-Parrinello 框架的数学形式

### 3.1 局部邻域的定义

对晶格上第 $i$ 个格点，定义一个以截断半径 $r_c$ 为界的局部邻域：

$$
\mathcal{C}_i = \{\Phi_j \mid |r_j - r_i| \le r_c\}
$$

**为什么需要这个公式**：我们需要精确规定"局部"到底指多大范围，这样才能把"只依赖局部环境"这个物理假设转化成一个可以直接实现的计算规则。

> **一句话直觉**：只收集半径 $r_c$ 以内所有邻居格点的信息，圈外的一律不看。

**逐项拆解**：

| 符号 | 含义 |
|------|------|
| $\Phi_j$ | 格点 $j$ 上的经典动力学变量（如局部晶格畸变、局部磁矩） |
| $r_j, r_i$ | 格点 $j$、$i$ 的空间位置 |
| $r_c$ | 截断半径，控制"局部"范围的大小，是模型的一个超参数 |
| $\mathcal{C}_i$ | 格点 $i$ 的局部环境——邻域内所有 $\Phi_j$ 的集合 |

**具体数值例子**：在本文将精读的论文中，方格子上取 $r_c = 3.61$ 个晶格常数，得到的局部邻域包含 45 个格点——这个数字后面会在等变神经网络的输入层维度中反复出现。

**为什么是这个形式**：截断半径 $r_c$ 太小会丢失重要的远程相互作用，太大则会让局部环境维度膨胀、计算代价增加。实践中 $r_c$ 通常取到"相互作用强度已经衰减到可忽略"的范围，是精度和效率之间的折中。

### 3.2 总能量的可加性分解

$$
E = \sum_i \epsilon_i = \sum_i \varepsilon(\mathcal{C}_i)
$$

**为什么需要这个公式**：我们要把"整个系统的能量"这样一个庞大、难以直接建模的量，拆成"每个格点各自贡献多少"的形式，这样才能用一个统一的、只处理局部信息的函数去逐个学习。

> **一句话直觉**：系统的总能量就是每个格点各自"应得的一份"加起来。

**逐项拆解**：

| 符号 | 含义 |
|------|------|
| $E$ | 系统总能量 |
| $\epsilon_i$ | 格点 $i$ 贡献的局部能量 |
| $\varepsilon(\cdot)$ | 一个通用函数（对所有格点都用同一个），把局部环境映射到局部能量 |
| $\mathcal{C}_i$ | 格点 $i$ 的局部邻域（如 3.1 节定义） |

**具体数值例子**：假设一个 3 格点的一维系统，$\varepsilon(\mathcal{C}_1)=2.1,\ \varepsilon(\mathcal{C}_2)=-0.5,\ \varepsilon(\mathcal{C}_3)=1.3$（单位：任意能量单位），则总能量

$$
E = 2.1 + (-0.5) + 1.3 = 2.9
$$

即使系统扩大到 300 个格点，只要每个格点局部环境的统计特性和训练时类似，同一个 $\varepsilon(\cdot)$ 函数依然适用，求和维度线性增长，不需要重新设计模型。

**为什么是这个形式**：能量的可加性本身是很多物理系统（尤其是短程相互作用占主导的系统）的自然性质；把它明确写成"逐格点求和"的形式，是让机器学习模型继承这一物理性质、并顺势获得线性扩展能力的关键设计。

### 3.3 局部能量的对称性要求

因为局部能量 $\epsilon_i$ 是一个标量（不带方向），物理定律要求：如果局部环境 $\mathcal{C}_i$ 经过某个对称操作 $R$ 变成 $\mathcal{C}_i'$，那么算出的局部能量必须完全相同——$\epsilon_i(\mathcal{C}_i') = \epsilon_i(\mathcal{C}_i)$。这正是[不变性](/前置知识/001n_前置知识_等变神经网络与不变性等变性)的要求。

BP 框架传统的做法是：先手工设计一组**对称不变描述符** $g=\{g_\ell\}$（比如原子间距离、角度的某些函数组合），保证 $g(\mathcal{C}_i') = g(\mathcal{C}_i)$，再用普通神经网络学习 $\epsilon_i = \varepsilon_{\text{ML}}(g(\mathcal{C}_i))$。因为 $g$ 本身就不变，网络输出自然也不变——这是"先设计不变特征，再套用普通网络"的经典思路。

### 3.4 从能量到力：直接学习力的替代方案

在绝热（Born-Oppenheimer）近似下，力可以由能量对局部变量求导得到（这就是 Hellmann-Feynman 定理）：

$$
F_i = -\frac{\partial \langle \hat H_e\rangle}{\partial \Phi_i} = -\mathrm{Tr}\Big[\hat\varrho_e\big(\{\Phi_i\}\big)\,\frac{\partial \hat H_e}{\partial \Phi_i}\Big]
$$

同样利用近视性原理，力也可以假设只依赖局部环境：

$$
F_i = \mathcal{F}(\mathcal{C}_i)
$$

**为什么需要这个公式**：如果只学能量再对它求导，训练目标和最终需要的物理量（力）之间隔了一层微分运算，误差会被放大；直接学习力可以跳过这个中间环节，让模型直接对齐真正需要的物理量。

> **一句话直觉**：既然最终要用力去驱动动力学模拟，不如让模型直接学"给定局部环境，力是多少"，不绕远路。

**逐项拆解**：

| 符号 | 含义 |
|------|------|
| $F_i$ | 格点 $i$ 上的力（可能是多分量的向量或张量，取决于 $\Phi_i$ 的类型） |
| $\mathcal{F}(\cdot)$ | 通用的力函数，对所有格点都用同一个模型 |
| $\mathcal{C}_i$ | 格点 $i$ 的局部环境 |

**为什么是这个形式**：但和局部能量不同，力**不是标量**——如果 $\Phi_i$ 本身带方向（比如晶格位移是一个二维向量），力也带方向，必须满足**等变性**而不是不变性：局部环境转多少，算出来的力也要跟着转多少。这正是传统"先设计不变描述符"的方法在预测力这类等变量时会遇到的根本困难——不变描述符从构造上就丢失了方向信息，无法直接告诉网络"力该指向哪个方向"。这也是本文将要精读的论文选择用等变神经网络（而不是传统不变量描述符）来直接预测力的核心动机。

## 四、BP 框架的两个关键优势总结

1. **线性扩展性**：因为能量/力被写成"逐格点求和 + 局部函数"的形式，且所有格点共用同一个模型，计算代价随系统尺寸 $N$ 线性增长，而不是随全局构型维度指数或超线性增长。
2. **可迁移性**：在小系统（比如 $40\times 40$ 格点）上训练出的局部模型，因为只处理固定大小的局部邻域，可以直接迁移到更大的系统（比如 $200\times 200$ 格点）上使用，不需要重新训练。

这两个优势正是本文要精读的论文能够做到"用小系统训练、在大系统上做非平衡动力学模拟"的技术基础；而该论文相对于传统 BP 框架的创新，在于把第二部分讲的"手工设计不变描述符"替换成了"等变神经网络直接处理带方向的局部环境"，从而能够直接、精确地预测带方向的力，而不必依赖对不变量求导这种间接方式。

## 五、总结

| 概念 | 一句话 |
|------|--------|
| 近视性原理 | 电子性质主要由局部环境决定，远处的贡献因波动干涉大多抵消 |
| 局部邻域 $\mathcal{C}_i$ | 半径 $r_c$ 内的所有邻居构型，是模型的输入单位 |
| 能量可加性分解 | $E=\sum_i \varepsilon(\mathcal{C}_i)$，把全局能量拆成逐格点局部贡献之和 |
| 对称性约束 | 局部能量（标量）需要不变性；局部力（矢量）需要等变性 |
| 线性扩展 | 因为模型只处理固定大小的局部输入，总计算量随系统尺寸线性增长 |
